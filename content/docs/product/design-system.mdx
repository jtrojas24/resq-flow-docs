---
title: Design system
description: The visual language, tokens, and shape rules that support resq-flow.
---

The current design direction is `Slate Refined · Ocean Blue`.

The design system exists to make the UI feel coherent across:

- flow home and metrics views
- the canvas and inspector
- logs and run detail
- status, timing, and semantic node families

## Core color tokens

### Dark mode

| Token | Value | Purpose |
| --- | --- | --- |
| `--surface-primary` | `#020617` | app background |
| `--surface-raised` | `#06101f` | panel and card surfaces |
| `--surface-overlay` | `#0a1e38` | hover and input surfaces |
| `--accent-primary` | `#42a5f5` | primary accent |
| `--border-default` | `rgba(30,58,95,0.7)` | standard borders |
| `--text-primary` | `#f1f5f9` | primary text |
| `--text-secondary` | `#4d7fa8` | secondary text |

## Typography

| Scale | Class | Use |
| --- | --- | --- |
| display | `text-2xl font-semibold` | stat values |
| title | `text-base font-medium` | section headings |
| body | `text-sm` | panel and log content |
| label | `text-xs` | badges, headers, filters |
| canvas | `text-[11px]` | node labels only |

Guidance:

- use `Inter` for normal UI text
- use monospace only for data such as trace IDs, timestamps, and raw values
- avoid tiny `text-[9px]` and `text-[10px]` UI text

## Canvas shape language

| Semantic role | Default shape |
| --- | --- |
| trigger | pill |
| queue | rounded rectangle |
| worker | rounded rectangle |
| scheduler or process | rounded rectangle |
| decision | diamond |
| resource | cylinder |
| visible detail | rounded rectangle inside a group when possible |
| note | annotation |

Rules:

- queues, workers, and processes should feel like one family
- use pills only for external starting conditions
- avoid turning the main graph into a badge wall

## Language rules

Prefer:

- `Flow`
- `Run`
- `Node`
- `Logs`
- `Status`
- `Timing`

Avoid using `trace`, `span`, `event`, and `telemetry` as the primary UI language outside advanced or raw detail views.
